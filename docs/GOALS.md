# Idle Game 2026 — 專案計畫書

## 1. 專案總結 (Project Overview)

**專案名稱：** Idle Game 2026
**倉庫：** `ochowei/idle-game-2026-2`
**專案類型：** 放置型 / 增量型遊戲 (Idle / Incremental Game)
**目標平台：** Web (瀏覽器)
**技術基礎：** 以 AI Studio (Gemini) 為輔助開發工具，前端為主的 Web 應用

### 專案簡介

本專案旨在開發一款基於瀏覽器的放置型遊戲 (Idle Game)。玩家透過點擊、升級、自動化等機制來累積資源與進度，即使離線時遊戲也能持續運行。專案將採用迭代式開發，從核心遊戲迴圈開始，逐步擴展至完整的遊戲體驗。

## 3. MVP 第一階段定義 (Phase 1 MVP Scope)

> **目標：** 一個可遊玩的最小完整遊戲迴圈 — 玩家可以點擊產出資源、購買生產者自動產出、購買升級強化產出、存檔與讀檔。

### MVP 包含範圍

| Epic | Feature | Tasks |
|------|---------|-------|
| Epic 1 | 1.1 專案初始化 | 1.1.1 ~ 1.1.5 |
| Epic 2 | 2.1 遊戲迴圈 | 2.1.1 |
| Epic 2 | 2.2 狀態管理 | 2.2.1 ~ 2.2.4 |
| Epic 2 | 2.3 大數值系統 | 2.3.1 ~ 2.3.2 |
| Epic 3 | 3.1 資源系統 | 3.1.1 ~ 3.1.2 |
| Epic 3 | 3.2 點擊與生產者 | 3.2.1 ~ 3.2.5 |
| Epic 3 | 3.3 升級系統 | 3.3.1 ~ 3.3.2 |
| Epic 4 | 4.1 主遊戲畫面 | 4.1.1 ~ 4.1.5 |

### MVP 不包含範圍（Phase 2+）

- 離線收益計算 (2.1.2)
- 匯出 / 匯入存檔 (2.2.5)
- 成就系統 (3.4)
- 輔助介面 (4.2)
- 響應式設計 (4.3)
- 轉生系統 (5.1)
- 自動化系統 (5.2)
- 挑戰模式 (5.3)
- 美術與音效 (Epic 6)
- 品質保證與優化 (Epic 7)
- CI/CD (1.2)

### MVP 驗收標準 (Acceptance Criteria)

1. 玩家可以透過點擊按鈕獲得主要資源
2. 玩家可以使用資源購買至少 3 種不同的生產者
3. 生產者每秒自動產出資源，數值即時更新於畫面
4. 玩家可以購買升級來強化點擊或生產者的產出
5. 購買成本隨數量遞增
6. 遊戲狀態自動存檔至 localStorage
7. 重新開啟頁面後能正確載入先前進度
8. 大數值能以易讀格式顯示（如 1.5M、3.2B）
9. UI 基本可用，資訊清晰易懂
